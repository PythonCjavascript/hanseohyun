# ⚾ KBO 타자 월별 성적 보정 및 시뮬레이션: 구자욱 선수 사례
### (KBO Hitter Monthly Performance Analysis & Correction Project)

## 📌 프로젝트 소개 (Overview)
이 프로젝트는 **데이터 전처리(Data Preprocessing)**가 데이터 분석의 신뢰도에 얼마나 큰 영향을 미치는지 보여주기 위해 기획되었습니다.

삼성 라이온즈의 대표 타자 **구자욱 선수**의 연도별/월별 기록을 분석 대상으로 삼았으며, 타수(AB)가 현저히 적은 달에 발생하는 **'평균의 함정(Small Sample Size Problem)'**을 통계적 기법으로 보정하여 선수의 진짜 기량을 파악하는 것을 목표로 합니다.

> *"전처리는 기계가 데이터를 빠르게 해석할 수 있도록 하는 첫 단계이며, 전체 분석 과정의 50~80%를 차지할 정도로 중요하다."* (참고: HDL_전처리_한서현.pdf)

## 🎯 문제 의식 (Problem Statement)
### "10타수 4안타(0.400)가 100타수 30안타(0.300)보다 정말 더 잘한 것일까?"

야구 데이터, 특히 월별 성적을 볼 때 우리는 종종 **착시 현상**을 겪습니다.
* **현상:** 특정 달에 타율이 4할이 넘는 맹타를 휘두르다가, 어떤 달은 1할대로 추락하는 등 기복이 심해 보임.
* **원인:** 부상 복귀, 시즌 초반/후반 등 **타석 수가 비정상적으로 적은 달**의 기록이 전체 평균을 왜곡함.
* **해결책:** 표본(타수)이 부족한 달의 데이터를 단순히 제외하는 것이 아니라, **'만약 정상적으로 타석에 들어섰다면?'**이라는 가정하에 데이터를 보정하고 시뮬레이션합니다.

## 🛠 분석 방법론 (Methodology)

### 1. 데이터 수집 및 정제 (Data Cleaning)
* **Data Source:** KBO 공식 기록 (구자욱 선수 데이터, `구자욱 기록.xlsx`)
* **Preprocessing:**
    * 누락된 연도(`Year`) 데이터를 `ffill()` (Forward Fill) 방식으로 채워 시계열 연속성 확보.
    * 분석에 불필요한 인덱스 컬럼 제거 및 재정렬.

### 2. 타수 보정 알고리즘 (Adjustment Logic)
통계적 왜곡을 줄이기 위해 다음과 같은 보정 로직을 설계했습니다.
1.  **월별 평균 타수(Average AB) 산출:** 해당 선수가 커리어 동안 각 월마다 평균적으로 소화한 타수를 계산합니다.
2.  **조건부 보정 (`modified_AB_if_smaller`):**
    * `실제 타수 < 월별 평균 타수` 인 경우: **월별 평균 타수**를 기준으로 설정.
    * 그 외의 경우: **실제 타수**를 그대로 사용.

### 3. 성적 시뮬레이션 (Simulation)
* 보정된 타수를 기반으로 **기대 타율(Simulated AVG)**과 **기대 안타 수(Simulated Hits)**를 재산출합니다.
* 이를 통해 "타수가 적어 운 좋게 타율이 높았던 달"이나 "타수가 적어 과소평가된 달"의 거품을 걷어내고 객관적인 지표를 생성합니다.

* **Language:** Python 3.x
* **Libraries:**
    * `Pandas`: 데이터 프레임 처리, 결측치 제어(`ffill`), 그룹화(`groupby`)

## 📊 결과 요약 (Insights)
시뮬레이션을 통해 재산출된 성적(`simulated_AVG`)과 실제 성적(`AVG`)을 비교한 결과:
1.  타수가 극단적으로 적었던 달의 **'뻥튀기된 타율'이 보정**되었습니다.
2.  이를 통해 선수의 월별 컨디션 사이클을 더 정확하게 파악할 수 있는 새로운 지표를 제시했습니다.
3.  단순 기록 너머의 **'지속 가능한 성적'**을 예측하는 데 기여합니다.
